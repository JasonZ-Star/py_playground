## 第四轮：模板选择器原生下拉弹层残留
- 用户反馈：截图显示点击"示例代码"下拉时，**浏览器原生蓝色下拉弹层与自定义白色 panel 同时出现**，重叠
- 根因：自定义下拉组件（trigger + panel）已实现，但原生 `<select id="template-selector">` 仍存在于 DOM 中。CSS 只用 `clip:rect(0,0,0,0); width:1px; height:1px` 做视觉隐藏，**没有 `display:none / visibility:hidden / pointer-events:none`**，也未设 `inert / tabindex=-1 / aria-hidden`。任何能让 select 接收 focus 或被点击的代码路径（`dispatchEvent('change')`、setter 钩子 `_setTemplateSelectorValue` 等）都会让浏览器弹出系统级下拉
- 修复（index.html）：
  - CSS 行 147：补充 `display:none !important; visibility:hidden !important; pointer-events:none !important; user-select:none`
  - JS setupTemplateSelector 开头（行 1651 后）：原生 select 加 `tabindex="-1"` + `aria-hidden="true"` + `aria-label` + `inert` 属性，并在 capture 阶段 swallow `mousedown/mouseup/click/keydown/keyup/focus/focusin` 事件（兜底某些 Chromium 仍然派发到 select 的情况）
- 验证：本机 `python -m http.server 5500` 已起，浏览器访问 http://localhost:5500/ 即可看到修复效果
- 注意：`inert` 不影响 JS 属性写入（`selector.value = ...` 仍生效）和自定义 `dispatchEvent('change')` — change 监听器在 select 自身上，触发不受影响

## 第五轮：分享代码模板状态同步优化
- 用户反馈：分享链接打开后，模板选择器显示异常（选中项显示为灰色禁用样式，截图显示 trigger 下方出现残留文字）
- 根因分析（三个问题）：
  1. **恢复路径不同步**：`sel.value = pending.tpl` 直接操作隐藏的 native select，未通过自定义 UI 的 `_setTemplateSelectorValue()` → trigger 文字不更新、下拉选项不高亮

- 修复内容：

  - `_setTemplateSelectorValue()` 增强：增加 key 有效性校验 + 调用 `renderOptions()` 同步下拉面板的 `[data-selected]` 高亮
  - CSS 行 163：compact 规则清空声明体，仅保留注释说明继承上方 !important 隐藏规则

## 第六轮：分享链接重构 — ShareStore 本地化存储
- 用户反馈：分享用的 LZString/base64 硬编码在 URL 中，代码长了 URL 会过长
- 方案：引入 `ShareStore` 类，localStorage 存储完整 payload + 短 ID，URL 只含短 ID
- ID 格式：`Date.now().toString(36) + 4位随机base36`（如 `m2x3k8a2`，~10 字符）
- 有效期：7 天（`EXPIRE_MS = 7*24*60*60*1000`），过期自动清理
- 最大条目：30 条，超出时按过期时间最旧的先删
- 流程：
  1. `shareCode()` → `ShareStore.save(payloadObj)` → 拿到短 ID → 构建 `/s/<id>` 或 `share.html?p=<id>`
  2. 若 localStorage 写入失败（配额满）→ 回退到 LZString 嵌入 URL（跨设备可用）
  3. 接收端 `tryDecodePayload()` 统一解码：先查 localStorage → 再试 LZString → 再试 raw JSON
  4. `/s/` 和 `?z=` 两条接收路径均改用 `tryDecodePayload()`
  5. 页面加载时调 `ShareStore.cleanup()` 清理过期条目
  6. 若 payload 存在但解码失败 → 显示「分享链接已过期或不在当前浏览器中」提示
- 分享弹窗增加 `#share-info` 行：本地分享显示有效期+「仅限当前浏览器」，LZString 显示「跨设备可用」

- 向后兼容：旧的 LZString/base64 链接仍可通过 tryDecodePayload 正常解码
